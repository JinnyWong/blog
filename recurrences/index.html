Recurrence

A recurrence is an equation that describes the running time of a recursive algorithm. The general form of a recurrence relation is:

an=f(an−1,an−2,...,an−k)a_n = f(a_{n-1},a_{n-2}, ..., a_{n-k})an​=f(an−1​,an−2​,...,an−k​)

For example, the running time of the merge sort algorithm can be described by the recurrence T(n)=2Tn2+O(n)T(n) = 2T\frac{n}{2} + O(n)T(n)=2T2n​+O(n), where T(n)T(n)T(n) is the running time of the algorithm on an input size of nnn. We can solve this recurrence to find the running time of the algorithm for any input size.

  • Arithmetic Sequence

an=an−1+da_n = a_{n-1} + dan​=an−1​+d

  • Geometric Sequence

an=r⋅an−1a_n = r \cdot a_{n-1}an​=r⋅an−1​

  • Fibonacci Sequence

Fn=Fn−1+Fn−2F_n = F_{n-1} + F_{n-2}Fn​=Fn−1​+Fn−2​

Methods use to solve recurrences:

  • Substitution Method
  • Recursion Tree Method
  • Master Theorem

Substitution Method

Substitution method solves recurrences by guessing the form of the solution and using mathematical induction to prove it.

Steps:

  1. Based on the recurrence relation, make an educated guess about the asymptotic form of the solution. This guess is often based on experience or patterns observed in simpler cases.
  2. Induction hypothesis: assume that the guessed form is correct for smaller values of kkk.
  3. Inductive step: Prove that if the guess is correct for smaller values, it also holds for the general case nnn.
  4. Verify that the solution holds for the initial conditions or base case.

Master Theorem

The master theorem is a general method for solving recurrences of the form

T(n)=aT(nb)+f(n)T(n) = aT (\frac{n}{b}) + f(n)T(n)=aT(bn​)+f(n)

where a≥1a \geq 1a≥1 and b>1b >1b>1

  • nnn : the size of the current problem
  • aaa : the number of subproblems in the recursion
  • nb\frac{n}{b}bn​ : the size of each subproblem
  • f(n)f(n)f(n) : the cost of the work that has to be done outside the recursive calls (cost of dividing + merging)

The Master Theorem gives asymptotic bounds for the solution to such recurrences by comparing f(n)f(n)f(n) with nlogban^{log_b a}nlogb​a:

  • Case 1: Running time is dominated by the cost at the leaves

T(n)=θ(nlogba)T(n) = \theta(n^{log_b a})T(n)=θ(nlogb​a)

  • Case 2: Running time is evenly distributed throughout the recursion tree
    • If f(n)=θ(nlogba)f(n) = \theta(n^{log_b a })f(n)=θ(nlogb​a)

T(n)=θ(nlogbalog(n))T(n) = \theta(n^{log_b a} log(n))T(n)=θ(nlogb​alog(n))

  • Case 3: Running time is dominated by the cost at the root
    • If f(n)=Ω(nlogba+ϵ)f(n) = \Omega(n^{log_b a + \epsilon})f(n)=Ω(nlogb​a+ϵ)

  1. Extract aaa, bbb and f(n)f(n)f(n) from a given recurrence.
